Ovumcy is the web app case, so the demo uses a real backend rather than a static export. The account is preloaded with sample cycle data, and the first segment tours the normal dashboard and calendar view to show what is already in the product before sharing.

From settings, SMART Link is still user directed: the browser shows the account scope, date range, and category choices. The server reads only the approved rows for that account and creates the reviewed snapshot. The active card shows the live QR and the same link used for copy, share, and open. The host does not receive plaintext; the backend encrypts the payload and uploads compact ciphertext to shlep with expiry and open limits.

The Chromium segment uses the newly created link in the cycle.fhir.me viewer. The key is in the URL fragment, so the viewer fetches ciphertext and decrypts locally before showing the clinician review. Revoking from Ovumcy posts the manage token and disables the old QR.
