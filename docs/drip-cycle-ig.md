# drip Cycle IG implementation journal

## App-specific checklist

| Item | drip implementation |
| --- | --- |
| Share review screen | Settings/Data management now has "Share Cycle IG data". drip is a local single-profile app, so there is no person/account selector. The user chooses date range and category switches, then previews counts before minting a live QR/link. |
| Layer 0 source | Stored `CycleDay.bleeding.value` becomes the core menstrual-bleeding fact. `bleeding.exclude !== true` maps to `valueBoolean: true`; `bleeding.exclude === true` maps to `valueBoolean: false` as a stored non-menstrual bleeding fact. Missing days and untouched defaults do not emit negative facts. |
| Layer 1 source | Flow maps from drip bleeding values 0-3 to spotting/light/moderate/heavy. Basal body temperature maps from `temperature.value` plus optional `temperature.time`. Pain and mood boolean selections map as symptom facts. Mucus, cervix, and notes are preserved with app-native coding/text. |
| Intentionally omitted | Sex, desire, predictions, derived cycle starts/phases, excluded temperatures, empty defaults, and absent rows. drip does not store numeric 0-10 pain scores, so it does not emit Numeric Pain Severity. |
| Live SHLink host | `https://shlep.exe.xyz` using shlep direct-file mode. The host receives compact JWE ciphertext only, enforces expiry, max-use, audit, and revoke. The app does not surface passcodes or access logs because this flow does not configure or display them. |
| QR handoff widget | One viewer-prefixed link, `https://cycle.fhir.me/view#shlink:/...`, is rendered as QR and used unchanged for copy, native share, and open-viewer controls. |
| Stop sharing | The app stores the shlep share id and manage token in component state and calls DELETE on Stop Sharing. Automatic stop conditions are seven days and five opens. |
| Receiver path | The app opens `https://cycle.fhir.me/view#shlink:/...`. In Chrome, the reference viewer populated the SHLink field; pressing its "Open link" button decrypted and rendered the Bundle client-side. |
| Privacy and validation evidence | Unit tests cover the Bundle shape, one-snapshot scoping, ciphertext-only upload, decrypt round trip, and omitted sensitive fields. Live smoke tests cover shlep create, resolve, decrypt, revoke, and max-use exhaustion. App QR evidence below confirms the real QR resolves to shlep ciphertext and decrypts locally. |

## Data flow

The share screen creates one immutable snapshot with `createCycleIgSnapshot`. The same snapshot feeds preview counts, normalization, encryption, shlep share creation, QR rendering, clipboard copy, native share, and open-viewer.

Sample data is generated by `lib/cycle-ig/sample-data.js` and loaded through drip's normal import path by `LoadCycleIgSampleData`, not by bypassing app storage. The verified sample run contained 55 cycle days from 2026-01-02 through 2026-06-22.

## Limitations

- No patient identity resource is emitted because drip's local data model does not provide a share-scoped patient/account identity.
- No passcode or access-log UI is shown. shlep supports those controls, but this drip flow only configures expiry, max-use, audit, and revoke.
- The reference viewer did not auto-render immediately from the fragment in Chrome during the emulator run; it populated the SHLink input and rendered after the viewer's "Open link" button was pressed.
